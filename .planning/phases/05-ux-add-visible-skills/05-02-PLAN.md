  <threat id="T-05-02-01" severity="high">UI reveals metadata of inaccessible skills through candidate rendering or disabled-state hints.</threat>
  <threat id="T-05-02-02" severity="medium">Client allows selecting already-added items and generates noisy duplicate mutation bursts.</threat>
  <threat id="T-05-02-03" severity="medium">Role-gating bug shows add controls to unauthorized users and creates confusing permission failures.</threat>

  <item>Render candidates only from visibility-safe hook output; no direct backend bypass calls from component.</item>
  <item>Mark existing members as disabled with explicit `Already in collection` badge and exclude from submission payload.</item>
  <item>Use existing owner/contributor/admin checks from collection detail state to guard action visibility.</item>
</mitigations>
