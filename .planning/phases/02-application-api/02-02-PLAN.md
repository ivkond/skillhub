      - `authenticated(HttpMethod.PATCH, "/api/web/collections/*")`
      - `authenticated(HttpMethod.DELETE, "/api/web/collections/*")`
      - `authenticated(HttpMethod.POST, "/api/web/collections")` — create
      - `authenticated(HttpMethod.POST, "/api/web/collections/*/skills")`
      - `authenticated(HttpMethod.DELETE, "/api/web/collections/*/skills/*")`
      - `authenticated(HttpMethod.PUT, "/api/web/collections/*/skills/order")`
      - `authenticated(HttpMethod.GET, "/api/web/collections/*/contributors")`
      - `authenticated(HttpMethod.POST, "/api/web/collections/*/contributors")`
      - `authenticated(HttpMethod.DELETE, "/api/web/collections/*/contributors/*")`
      - `authenticated(HttpMethod.PATCH, "/api/web/collections/*/visibility")`

    If `/api/v1` duplicates exist on controllers, add the **same** pattern set with `/api/v1/...` prefixes.

    Adjust segment counts (`*` vs `**`) to match **actual** `@GetMapping`/`@PostMapping` paths in `SkillCollectionController` / `PublicSkillCollectionController` if they differ (executor must align registry ↔ controller).

    - For **read-only** GETs that should work with existing read scopes: mirror `ApiTokenPolicy.allow(HttpMethod.GET, "/api/web/skills/**")` style — use `allow(HttpMethod.GET, "/api/web/public/collections/**")` if public catalog should be token-accessible; else `unsupported` by omission.
