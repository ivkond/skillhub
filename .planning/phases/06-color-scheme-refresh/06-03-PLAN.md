    2) `cd web && pnpm run test:e2e -- e2e/collections-flow.spec.ts`.
    3) Create `.planning/phases/06-color-scheme-refresh/06-PARITY-MATRIX.md` with screen rows:
       `landing`, `dashboard`, `collection detail`, `skill detail`, `notifications`, `login`, `register`, `reset password`.
       For each row include state columns: `default`, `hover`, `focus`, `disabled`, `invalid`, `loading`, `empty`, `dialog/dropdown`.
       Every evidence row must include strict fields:
       `screen`, `state`, `theme`, `steps`, `expected`, `actual`, `artifact_path`, `owner`, `timestamp`, `result`, `na_reason`.
       `result` accepts only `PASS`, `FAIL`, `N/A`.
    4) Add machine validator `web/scripts/check-parity-matrix.mjs` and package script `check:parity-matrix`:
       - parse `06-PARITY-MATRIX.md`,
       - enforce required fields and allowed `result` enum,
       - fail when `result=PASS` and referenced `artifact_path` does not exist,
       - fail when `result=N/A` and `na_reason` is empty,
       - fail when mandatory fields are empty or malformed.
       Command contract:
       `cd web && pnpm run check:parity-matrix -- --matrix ../.planning/phases/06-color-scheme-refresh/06-PARITY-MATRIX.md --artifacts-root .`.
    5) Apply auditable decision rules:
       - `PASS` only when `actual` matches `expected` and `artifact_path` exists;
       - `N/A` allowed only when state is not applicable for the screen, with mandatory `na_reason` and reproducible `steps`;
       - empty/missing field automatically counts as `FAIL`.
    6) Run explicit accessibility gate in both themes via `web/e2e/theme-accessibility.spec.ts`:
       - contrast checks for critical text/surface pairs on `landing`, `dashboard`, `collection detail`, `notifications`, `login` must satisfy WCAG AA (normal text >= 4.5:1, large text >= 3.0:1),
       - keyboard traversal (`Tab`, `Shift+Tab`, `Enter`, `Space`, `Escape`) reaches and operates primary interactive controls without trap,
       - focus indicator is visible (`:focus-visible` driven outline/ring present) for keyboard path controls.
    7) If any row is `FAIL` or unresolved `N/A`, or accessibility gate fails, fix regressions in this plan and rerun verification.
    8) Only when all rows are `PASS` or justified `N/A`, set `THEME_TOGGLE_RELEASED = true` in `web/src/shared/theme/theme-release.ts`.
       This flag is UX release visibility only and must not gate security/policy logic.
  </action>
