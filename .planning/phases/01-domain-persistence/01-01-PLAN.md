    **Table `skill_collection`** (root): columns at minimum `id BIGSERIAL PRIMARY KEY`, `owner_id VARCHAR(128) NOT NULL REFERENCES user_account(id)`, `slug VARCHAR(64) NOT NULL`, `title VARCHAR(200) NOT NULL`, `description TEXT NULL`, `visibility VARCHAR(20) NOT NULL` (store `PUBLIC`/`PRIVATE` strings to match `SkillVisibility`), `created_at`, `updated_at` as `TIMESTAMPTZ NOT NULL` with defaults consistent with nearby migrations, audit `created_by`/`updated_by` **only if** existing domain entities for collections will map them—otherwise omit to match minimal `Skill`-style columns (executor: align with chosen entity fields; NOT NULL timestamps required).

    **Table `skill_collection_member`**: `id BIGSERIAL PRIMARY KEY`, `skill_collection_id BIGINT NOT NULL REFERENCES skill_collection(id) ON DELETE CASCADE` (**D-03**, **COL-03**), `skill_id BIGINT NOT NULL REFERENCES skill(id)` (ON DELETE behavior: use **RESTRICT** or **NO ACTION** unless product requires CASCADE delete of memberships when skill deleted—default **RESTRICT** to avoid silent data loss), `sort_order INTEGER NOT NULL`, created/updated timestamps as for other join-style tables if the entity maps them. `UNIQUE (skill_collection_id, skill_id)` (**D-04**, **INT-02**).

    **Table `skill_collection_contributor`**: `id BIGSERIAL PRIMARY KEY`, `skill_collection_id BIGINT NOT NULL REFERENCES skill_collection(id) ON DELETE CASCADE` (**D-03**), `user_id VARCHAR(128) NOT NULL REFERENCES user_account(id)`, timestamps. `UNIQUE (skill_collection_id, user_id)` for idempotent contributor grants (research **A2**). No column storing “owner role” on contributors (**D-02**).

    Add supporting indexes: btree on `(skill_collection_id)` on both child tables; optional `(owner_id)` on `skill_collection` for cap queries in plan 01-02.

    Do **not** add HTTP routes or Phase-2-only columns. No share-token column (**D-07**).
